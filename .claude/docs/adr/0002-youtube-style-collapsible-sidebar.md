# 0002. YouTube風の折りたたみ可能な左サイドバーを採用

## Status

Accepted

## Date

2026-04-13

## Context

従来、認証後レイアウトには上部固定のトップナビゲーション (`NavBar`) を配置していた。モバイルではハンバーガー + Sheet ドロワー、デスクトップでは横並びリンクで切り替えていたが、以下の課題があった。

- メニュー項目が増えると（レシート / レポート / プロジェクト / 顧客 / 担当者 / ユーザー管理 / 設定）横並びで窮屈になり、将来の拡張余地がない
- トップバーは縦スペースを常に消費し、レシート明細テーブルなどコンテンツ量の多い画面で作業領域を圧迫する
- 「今どのセクションにいるか」の視認性がロゴ横のテキストリンクだけでは弱く、YouTube / Gmail / Linear など一般的な業務系アプリに慣れたユーザーから見て直感性に欠ける
- RBAC（`admin` / `editor` / `viewer`）に応じて出し分ける項目が増えるほど、横並びでは余白設計が難しい

代替案として検討したもの:

1. **現状維持（トップバー）**: 実装コスト最小だが上記課題を解決しない
2. **常に固定幅の左サイドバー（折りたたみ不可）**: 縦スペースは解放されるが、横スペースを常に消費し、レシート一覧の横幅を奪う
3. **YouTube風の折りたたみ可能サイドバー（採用）**: デスクトップでは `w-60` / `w-16` をトグル、モバイルでは Sheet ドロワーで表示。トップにはロゴと折りたたみボタンのみ置いた薄いヘッダーを残す

## Decision

認証後レイアウト（`src/app/(authenticated)/layout.tsx`）のナビゲーションを、`NavBar` コンポーネントから `AppShell` コンポーネントへ差し替える。

- **デスクトップ (`md` 以上)**: `position: fixed` の `<aside>` を左側に配置。`collapsed` state により `w-60`（ラベル表示）と `w-16`（アイコンのみ）を切り替える。メインコンテンツ側は `md:pl-60` / `md:pl-16` でサイドバー幅分のパディングを動的に適用
- **モバイル (`md` 未満)**: サイドバーを `hidden` にし、トップヘッダー左のハンバーガーアイコンから Sheet ドロワーで同一のナビゲーションを提供
- **RBAC**: `allNavItems` に `roles?: UserRole[]` を持たせ、`tksUser.role` で絞り込む既存方式を踏襲
- **アクセシビリティ**: `<aside>` と `<nav>` に `aria-label`、折りたたみボタンに `aria-expanded` / `aria-controls`、アクティブリンクに `aria-current="page"`、折りたたみ時のアイコンリンクに `aria-label` を付与

## Consequences

### Positive

- 縦スペースが解放され、レシート一覧など縦に長いテーブルの可視領域が増える
- メニュー項目が今後増えても縦方向にスケールする
- 折りたたみ時は `w-16` まで圧縮でき、横スペースを奪う度合いを最小化できる
- アクティブページの視覚的手がかり（ハイライト + `aria-current`）が明示され、現在位置の認知負荷が下がる
- ロール別メニュー出し分けが縦並びの行単位になり、差分が見やすい

### Negative

- `AppShell` 内でデスクトップサイドバーとモバイル Sheet ナビのリンクレンダリングがほぼ同じロジックで並存する（アイコンサイズ・折りたたみ対応の違いのみ）。将来メニュー構造が複雑化したらサブコンポーネント抽出が必要になる可能性
- `collapsed` state は `useState(false)` 初期値のみでブラウザリロード後の永続化をしていない。ユーザーごとの好みを保持するには `localStorage` 連携が別途必要
- ファイル行数は `NavBar` (176行) から `AppShell` (245行) に増え、単一コンポーネントの責務がやや肥大化（デスクトップ固定サイドバー + トップヘッダー + モバイル Sheet を1ファイルに同居）

## Compliance

- PR マージ後に `src/app/(authenticated)/components/NavBar.tsx` が存在しないことを確認（本 PR で削除済み）
- 認証後の任意ページ（`/receipts`, `/reports`, `/settings` など）でサイドバーが表示され、折りたたみボタンが `aria-expanded` を切り替えることを手動確認
- モバイル幅（`< md`）でハンバーガーメニューから同一項目が表示されることを手動確認
- RBAC: `viewer` ロールで `/projects` `/clients` `/staff` `/settings` `/settings/users` がサイドバーに表示されないことを確認
